# Moloko

АИС контроля качества и учета производственных партий молочной продукции для КФХ Жукушева К.Н.

## Что реализовано

- учет поступления сырого молока и создание партий;
- карточка партии с остатком, статусом, сроком годности и историей операций;
- лабораторный контроль с автоматическим заключением по нормам;
- блокировка партий с критическими отклонениями;
- запрет отгрузки заблокированных партий и объема больше остатка;
- складские операции: поступление, перемещение, списание;
- оформление отгрузок и подготовка данных для ручного переноса в ФГИС "Меркурий";
- справочники пользователей, групп животных, емкостей, покупателей и транспорта;
- журнал аудита;
- CSV-экспорт и резервное копирование;
- подключение к PostgreSQL через Npgsql.

## Подключение PostgreSQL

Настройки лежат в файле:

```text
Moloko/database.json
```

По умолчанию используется строка:

```text
Host=localhost;Port=5432;Database=moloko;Username=postgres;Password=postgres
```

Если у базы другой пользователь, пароль или имя, поменяйте `ConnectionString`.
При запуске приложение проверяет схему из `Moloko/Database/postgresql_schema.sql`.
Если база пустая, будут добавлены начальные справочники и тестовая партия.

Если PostgreSQL временно недоступен, приложение переключится на локальный JSON и покажет причину в верхней строке статуса.

## Запуск

```powershell
dotnet run --project Moloko\Moloko.csproj
```

## Файлы данных

CSV-отчеты и резервные копии создаются здесь:

```text
%LOCALAPPDATA%\Moloko\Exports

```
